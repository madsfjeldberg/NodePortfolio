What is REST API?

REST (Representational State Transfer) is an architectural style for designing networked applications. RESTful APIs use HTTP requests to perform CRUD operations (Create, Read, Update, Delete).
It works by allowing clients to send requests to the server, the server processes the request and sends a response back to the client, typically in JSON format.

REST API principles

  • Using the right HTTP method
  • Ordering
  • Naming/mapping to collections

The right HTTP method

The right HTTP method should be used for the right operation.
For example, if you want to create a new resource, you should use the POST method.
If you want to retrieve a resource, you should use the GET method.
If you want to update a resource, you should use the PUT method.
And, of course, if you want to delete a resource, you should use the DELETE method.

Ordering

When designing a REST API, the methods should be in the following order:

  • GET - Retrieve resources
  • POST - Create new resources
  • PUT - Update existing resources
  • DELETE - Remove resources

This ensures that the API is consistent and easy to understand, which is important when larger teams are collaborating on the same API.

Naming/mapping to collections

When naming/mapping to collections, the plural form of the resource should be used.
It is also recommended to keep the endpoints short and concise.

Good:

  • GET /api/users
  • POST /api/giraffes
  • DELETE /api/wrestlers/1

Bad:

  • GET /api/all_users
  • GET /api/user
  • POST /api/giraffe
  • DELETE /api/wrestler/1

Example - using Express

A very basic example of a REST API using Express.

Note that the database functions are not defined in this example, it's just pseudocode to give an idea of how the API works. 

// GET /api/users
app.get('/api/users', (req, res) => {
  let data = database.getUsers();
  res.json({ data: data });
});

// POST /api/users
// it's good practice to add a comment above the function
// to describe what it does, and add the parameters and return value.
// this is just an example though.
/**
 * @param {Object} req.body - The user data to create.
 * @param {string} req.body.name - The name of the user.
 * @param {string} req.body.email - The email of the user.
 * @returns {Object} The created user.
 */
app.post('/api/users', (req, res) => {
  let data = database.createUser(req.body);
  res.json({ data: data });
});

// PUT /api/users/1
app.put('/api/users/:id', (req, res) => {
  let data = database.updateUser(req.params.id, req.body);
  res.json({ data: data });
});

// DELETE /api/users/1  
app.delete('/api/users/:id', (req, res) => {
  let data = database.deleteUser(req.params.id);
  res.json({ data: data });
});
